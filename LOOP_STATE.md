## Architecture (planned end state)

```
push to a PR
   â”‚
   â–Ľ
Codex reviews the PR        (GitHub App, automatic on every push â€” not a workflow here)
   â”‚  leaves P1/P2 findings
   â–Ľ
Bridge (codex-auto-fix.yml) posts ONE "@claude fix" per review wave   (via AUTOMATION_PAT)
   â”‚
   â–Ľ
Claude Fixer (claude.yml)   fixes on a branch â€” the RELIABLE fixer.
   â”‚  opens a PR with "Fixes #N" + the `automerge` label
   â–Ľ
Codex re-reviews the new commit
   â”‚  Circuit breaker: after 3 @claude-fix rounds that don't converge
   â”‚  â†’ add `needs-owner` + Telegram ping, stop auto-triggering
   â–Ľ
Merge Bot (merge-bot.yml)   merges when everything is green
                            (identifies Claude PRs by the `automerge` LABEL, not author login,
                             because Claude's PRs are PAT-authored = owner, not a bot)
```

**Direction is deliberate: Codex reviews, Claude fixes.** Codex is unreliable at *fixing* (phantom commits); Claude is the trusted fixer. We never hand the fix back to Codex.

**Labels (uniform across all repos):**
| Label | Meaning |
|-------|---------|
| `claude-fix` | "Claude, fix this." Set by CI Doctor; triggers Claude Fixer. |
| `automerge` | May be auto-merged by Merge Bot once green. Set by Claude's PR step. |
| `needs-owner` | Escalation â€” automation stopped, a human must act. Hard stop for Merge Bot. (The sole escalation label; the older one was fully removed.) |
| `ci-doctor` | Marks Issues opened by CI Doctor (dedup + close). |

## Key decisions made

- **Claude fixes, Codex reviews.** Codex produces phantom/empty fix commits; Claude is the reliable fixer.
- **`--max-turns 20`** in claude.yml (balance cost vs completing the task; most fixes need far fewer than 35).
- **`cancel-in-progress: false`** on claude.yml â€” never kill a run that is already burning money; a second event queues behind it.
- **Final minimal allowlist** in claude.yml (`Read,Glob,Grep,Edit,Write` + scoped `git`/`gh pr`/`gh issue`/`actionlint`; no bare `Bash`, no interpreters, no `gh api`) â€” ends the "shrink-the-allowlist" loop; if something's missing the fixer fails to `needs-owner` rather than looping.
- **Codex Gate kept as a blocking check** â€” the human waits for the loop instead of merging manually.
- **Merge Bot identifies Claude PRs by the `automerge` label OR a same-repo `claude/*` branch**, not author login (Claude's PRs are PAT-authored = owner `funzi7`, not a bot login). The escalation hard-stop is always checked first, and the protected-path guard still runs before any merge. **Merge Bot evaluates only the LATEST check run per name** (GitHub creates a fresh `check-codex-status` run on each gate pass; the old early-red run must be ignored) â€” this was why merge-bot never merged.
- **Cost:** ~$1â€“1.7 per Claude fix run (duration-based). A Spending Limit is set in the Anthropic Console.
- **Escalation label standardized on `needs-owner` (older label fully removed).** Every workflow now adds AND checks only `needs-owner`. The earlier backward-compat dual-matching (the `LABEL_ESCALATE_LEGACY` constant and the `|| ... LEGACY` clauses in merge-bot's candidate hard-stop and ci-doctor's "already escalated" skip) has been deleted â€” there were no open issues/PRs still carrying the old label, so the compat shim was unnecessary complexity. `needs-owner` is the single source of truth.
